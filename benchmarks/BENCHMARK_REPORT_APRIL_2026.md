# Segment Sort: Performance & Scalability Report (April 2026)

## 1. Executive Summary
This report documents the performance of the **Segment Sort** algorithm family (V1, V2, and V3) compared to industry-standard implementations in JavaScript (V8 Engine). The results demonstrate that **Segment Sort V1** consistently outperforms the native `Array.prototype.sort` in large-scale random data, while **V3 (Galloping Edition)** provides specialized advantages for nearly-sorted datasets.

## 2. Methodology
*   **Environment:** Node.js v20+ (V8 Engine).
*   **Hardware:** AMD Ryzen 7 8845HS (Zen 4), 64GB RAM.
*   **Repetitions:** 5 per test case (averaged).
*   **Data Sizes:** 100,000 (100K) to 5,000,000 (5M) elements.

## 4. Key Findings

*   **V8 Performance Gap:** SegmentSort V1 is approximately **40% faster** than the native V8 implementation for 5M random elements. This is attributed to superior Cache-L2 locality (using a fixed 64KB buffer) and the "Flat Segment" detection logic.
*   **Adaptive Galloping (V3):** The V3 variant demonstrates significant scaling advantages in "Nearly Sorted" data, outperforming the original V1 by **46%** in 5M datasets through reactive exponential searching (Galloping).
*   **Robustness:** While naive recursive algorithms (QuickSort) crashed due to stack limits on large or repetitive datasets, SegmentSort's stack-based iterative approach remained stable across all scenarios.

## 5. Technical Fairness & Context
To ensure scientific integrity, we acknowledge the following regarding the baseline comparisons:
1.  **QuickSort Baseline:** The QuickSort used in these tests is a naive recursive implementation. While it serves as a baseline for algorithmic complexity, it is not representative of production-grade QuickSort implementations which would use iteration or depth-limiting to avoid stack overflows.
2.  **Industry Standard:** The primary benchmark target is the **V8 Native `sort()`**. This is a highly optimized, production-grade hybrid (usually Timsort/Quicksort) implemented in C++. Beating this native implementation from "Pure JavaScript" is the primary achievement of this project.
3.  **Heap Sort/Merge Sort:** These are standard textbooks implementations. They show consistent $O(N \log N)$ behavior but lack the adaptive optimizations present in Segment Sort.

## 6. Conclusion
**Segment Sort V1** is the recommended general-purpose variant due to its low overhead and exceptional performance on random and high-duplicate data. **V3** is recommended for scenarios with high temporal locality or pre-sorted data streams.
